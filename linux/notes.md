Решил поставить на свой вирт сервер баовую защиту от Linux UFW включается он одной командой, но перед этим нужно здаать ему чтобы он закрыл все порты кроме 22 на котором сидим мы, если не сделаем это то он нас выкинет с сервера и заблоикрует. Делается это так сначала sudo ufw allow 22/tcp задали порт который он не закрывает, sudo ufw enable - включаем брандмауэр, проверяем статус работы sudo ufw status. 
В ответе должно быть указано active и порт 22. Далее настроил контейниризацию(Docker) он упрощает работу несколких программ одновременно ведь он их упаковывает в свои личные контейнеры со всеми зависимостями и хранит отдельно от системы, то есть приложения с разными требованиями конфликтовать не будут, а когда нужно запустит сразу много контейнеров используем docker compose. Устанавливается это так одной командой docker и плагин compose - sudo apt install docker.io docker-compose-v2 -y. Далее прописываем условие чтобы каждый раз не писать sudo - супер-права администратора выдадим своему юзеру супер-права администратора делается это так sudo usermod -aG docker $USER .
Чтобы система поняла что ты оновил права нужно обновиит сессию командой - newgrp docker .
Проверяем работу контейнеров, запустим маленький текстовый контейнер docker run hello-world  . Он будет искать образ на сервере но его нету, не найдет пойдет в интренет в склад docker hub скачает нужную версию под мой процессор, распакует контейнер, запустит его и выведет приветственное сообщение и завершит работу контейнера. Он вывел " Hello from Docker"


Далее решил поиграться и поднять полноценную панель управления 3x-ui(граф.интерфейс для управления сервером Xray-core и протоколами VLESS/Reality. Создадим папку внутри docker - mkdir ~/3x-ui-lab и зайдем в нее - cd ~/3x-ui-lab .
Далее создал конфигурации Docker Compose решил упростить задачу и прописать сразу всю инфраструктуру сервиса в файле yaml - nano docker-compose.yml 
вставил туда конфиг: 
services:
  x-ui:
    image: ghcr.io/mhsanaei/3x-ui:latest
    container_name: 3x-ui
    volumes:
      - $PWD/db/:/etc/x-ui/
      - $PWD/cert/:/root/cert/
    environment:
      XRAY_VMESS_AEAD_FORCED: "false"
    network_mode: host
    restart: unless-stopped
Настроил UFW - sudo ufw allow 2053/tcp задал что будем работать на 2053 порту, это чтобы он не блокировал этот порт. Запускаем командой docker compose up -d  docker начал скачивать ядро  Xray и саму панель, чтобы проверить результат в адресной строке ввел http://192.168.64.2:2053 открылась панель 3x-ui ввел данные логин и пароль (admin) и зашел в панель и там все показано о системе и меню настройки xray.
